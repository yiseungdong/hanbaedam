## 작업 규칙 (중요)
1. 모든 새 기능은 먼저 의논 → 아이디어 제시 → 사용자 OK → 프롬프트 작성
2. 프롬프트는 항상 Part 1 / Part 2로 분리
3. git 명령어는 절대 && 로 연결하지 말 것 (항상 3줄로 분리)
4. 새 채팅 시작 시 첫 메시지: "hanbaedam-masterplan.md 읽어줘"
5. 채팅창이 차기 전에 마스터플랜 업데이트 후 저장
